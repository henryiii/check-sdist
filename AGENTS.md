# AGENTS.md

`check-sdist` builds an SDist and compares its contents against the files
tracked by git, reporting files that are in one but not the other. It optionally
injects common junk files (pycache, OS files) to verify the build backend
excludes them.

## Commands

This is a `nox` + `uv` project. Sessions are defined in `noxfile.py` (itself a
`uv run --script` file).

- `uv run nox -s tests` — run the full test suite (uses `pytest -nauto`)
- `uv run nox -s lint` — run prek (pre-commit) on all files
- `uv run nox -s pylint` — run pylint against `src` (slower; installs the
  package)
- `uv run nox -s minimums` — install lowest-direct dependency versions on 3.9
  and test
- `uv run nox -s coverage` — tests with coverage reporting
- `uv run nox -s build` — build SDist + wheel
- `uv run nox -s generate_schema` — regenerate
  `src/check_sdist/resources/check-sdist.schema.json`
- `uv run nox -s bump -- <args>` — bump the version (bump-my-version)

Run a single test directly: `uv run pytest tests/test_backends.py -k test_name`.
Pass pytest args after `--` to nox sessions, e.g.
`uv run nox -s tests -- -k inject`.

Lint with `prek -a --quiet` (not `pre-commit run -a`).

The package itself can be run on this very repo: `uv run check-sdist`.

## Architecture

The pipeline lives in `__main__.py::compare()`, which assembles two sets of
relative file paths and diffs them:

1. **SDist side** — `sdist.py::sdist_files()` shells out to `build` or
   `uv build` (selected by `select_installer`, honoring `--installer` and
   `--no-isolation`), unpacks the resulting `.tar.gz` to a temp dir, and returns
   the file names with the top-level package-version prefix stripped. `PKG-INFO`
   is always dropped.
2. **Git side** — `git.py::git_files()` runs `git ls-files --cached` (with
   `--recurse-submodules` by default). Alternatively `mode = "all"` walks the
   filesystem instead of git.

Differences are filtered through several ignore layers before being reported:

- **User config** from `[tool.check-sdist]` in `pyproject.toml`: `sdist-only`
  and `git-only` are gitignore-style pattern lists applied via
  `pathspec.GitIgnoreSpec`.
- **Default ignores** (when `default-ignore` is true): `git-only` patterns from
  `resources/default-ignore.txt`, plus `*.dist-info` (always) and the resolved
  backend's `sdist_only_ignores()` — setuptools `*.egg-info`/`setup.cfg` and the
  autogenerated version-file path for that backend (setuptools-scm, hatch-vcs,
  pdm, scikit-build-core).
- **Backend exclude lists** — the resolved backend's `git_only_excludes()` reads
  its exclude list and removes those from the git-only set. Hatchling and
  scikit-build-core use gitignore-style matching; the rest use glob matching.

Per-backend logic lives in the `backends/` package: one module per backend
(`setuptools.py`, `flit.py`, `hatchling.py`, …), each a `Backend` subclass
(`backends/_base.py`) owning both `git_only_excludes()` and
`sdist_only_ignores()`. Backends are registered under the `check_sdist.backends`
entry-point group (keyed by build-backend string) and discovered via
`backends/__init__.py::load_backends()`; `resolve_backend()` maps the
`build-backend` config (`auto`/`none`/explicit) to a `Backend` or `None`. Users
can register their own backends under the same entry-point group.

`compare()` returns a bitfield exit code: 1 = SDist has extra files, 2 = SDist
is missing git files, 3 = both.

The config schema is validated via the `validate-pyproject` plugin entry point
(`schema.py::get_schema`), which serves the committed JSON in `resources/`. That
JSON is generated from the YAML embedded in `scripts/generate_schema.py` — edit
the script and regenerate, don't hand-edit the JSON.

`inject.py` provides the `--inject-junk` context manager that temporarily
creates files listed in `resources/junk-paths.txt` and removes them (and any
directories it created) afterward.

## Conventions

- Every module declares `__lazy_modules__` listing its imports — a marker
  convention used by the project's lazy-import tooling; keep it in sync when
  changing imports.
- Modules avoid `typing.TYPE_CHECKING`; instead they set `TYPE_CHECKING = False`
  locally (this is enforced by a ruff banned-api rule).
- Python 3.9+ with `from __future__ import annotations` required in every file
  (ruff isort enforces this). mypy runs in strict mode; `check_sdist.*`
  additionally requires fully typed defs.
- ruff selects `ALL`; see `pyproject.toml` for the ignore list before
  suppressing inline.

## Pre-commit hooks

This repo ships its own hooks (`.pre-commit-hooks.yaml`): `check-sdist` (needs
build deps in `additional_dependencies`) and `check-sdist-isolated` (builds in
isolation, defaults uv into deps). `README.md` documents the consumer-facing
usage.
